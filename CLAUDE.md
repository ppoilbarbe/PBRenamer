# Project: PBRenamer
GUI batch file renamer. Python 3.12+, PySide6 (Qt6), src layout.
Publisher: PBMou | Author: Marcel Spock <mrspock@cardolan.net> | License: GPLv3

## Stack
- GUI: PySide6; Qt Designer — `make designer`
- Tests: pytest + pytest-qt (`qtbot`; never instantiate `QApplication` manually)
- Docs: Sphinx + sphinx-rtd-theme (ReadTheDocs)
- Lint: ruff (line-length 88, target py312)
- Env: conda `pbrenamer`, **conda-forge only** (`nodefaults`) — `make venv`
- Packaging: PyInstaller (`pbrenamer.spec`) — `make dist`
- CI/CD: `.github/workflows/ci.yml` — test → hooks → build (3 OS) → release on semver tag

## Key conventions
- All OS-specific code goes in `src/pbrenamer/platform/`. Never import `xdg.py` directly.
- `*_ui.py` are **gitignored** generated artifacts — never commit them; CI regenerates via `pyside6-uic`.
- Conflict detection in `_refresh_conflicts()` uses `platform.fs.conflict_key()` and `platform.fs.same_file_path()` for case-insensitive filesystem support.
- `make dist` calls `translate` first; CI runs `pyside6-uic` then PyInstaller directly (`.mo` files are in git).

## Translations

Catalogues: `src/pbrenamer/locale/{en,fr}/LC_MESSAGES/pbrenamer.{po,mo}` — both committed.
Python strings: wrap with `_("…")`. UI strings (`text`, `toolTip`, `statusTip`, `placeholderText`): set in `main_window.ui`, extracted automatically.

**After any user-visible string change:**
1. If `.ui` changed: `make ui`
2. `make translate` — updates `.pot`, merges `.po`, compiles `.mo`
3. Fill every new `msgstr ""` in `en` and `fr` `.po` files (keyboard shortcuts stay empty)
4. `make translate` again to compile the filled entries
5. Verify: `msgfmt --statistics locale/fr/.../pbrenamer.po -o /dev/null` — only shortcut msgids should remain untranslated

`msgstr ""\n"continued…"` is multi-line syntax, not an empty string. A truly empty msgstr is followed by a blank line with nothing after the `""`.

New language: `make new-lang LOCALE=de`, translate all `msgstr`, then `make translate`.

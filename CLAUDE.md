# Project: PBRenamer
GUI batch file renamer. Python 3.12+, PySide6 (Qt6), src layout.
Publisher: PBMou | Author: Marcel Spock <mrspock@cardolan.net> | License: GPLv3

## Stack
- GUI: PySide6; Qt Designer via `pyside6-designer` / `make designer`
- Tests: pytest + pytest-qt (use `qtbot`, never instantiate `QApplication` manually)
- Docs: Sphinx + sphinx-rtd-theme, hosted on ReadTheDocs
- Lint: ruff (line-length 88, target py312)
- Env: conda, env name `pbrenamer`, **conda-forge only** (`nodefaults`) — `make venv` to create
- Packaging: PyInstaller via `pbrenamer.spec` — `make dist` builds for the current platform
- CI/CD: GitHub Actions (`.github/workflows/ci.yml`) — test → hooks → build (3 OS) → release on semver tag

## Layout

```
src/pbrenamer/
├── __init__.py          version, author metadata
├── __main__.py          CLI entry point (--help / --version, then launches GUI)
├── i18n.py              gettext bootstrap; reads language from platform.dirs
├── xdg.py               compat shim → platform.dirs.XdgDirs (do not import directly)
├── core/
│   ├── filetools.py     file listing, text transforms, rename engine
│   └── undo.py          batch undo stack
├── platform/            all OS-specific code lives here
│   ├── dirs.py          AppDirs factory: XdgDirs (Linux) / _MacDirs / _WindowsDirs
│   ├── fs.py            is_case_sensitive(), same_file_path(), conflict_key()
│   └── locale.py        system_language() — env vars + locale.getlocale() fallback
└── ui/
    ├── main_window.py   main window (business logic)
    ├── main_window.ui   Qt Designer source (edit this, not the generated file)
    ├── main_window_ui.py  generated by pyside6-uic — gitignored, do not edit
    ├── presets.py       pattern preset persistence (uses platform.dirs)
    └── settings_dialog.py  language settings

tools/
├── extract_ui_strings.py   pulls translatable strings from *_ui.py for xgettext
└── extract_changelog.py    reads CHANGELOG.md entry for a given tag (used by CI)

.github/workflows/ci.yml    CI pipeline (test + hooks + 3-platform build + release)
pbrenamer.spec              PyInstaller spec (version + platform embedded in artifact name)
environment.yml             conda env — conda-forge only, nodefaults
```

## Key conventions
- All new OS-specific code goes in `src/pbrenamer/platform/`. Never import `xdg.py` directly.
- `*_ui.py` files are **gitignored** generated artifacts — never commit them; CI regenerates them via `pyside6-uic` before building.
- Conflict detection in `_refresh_conflicts()` uses `platform.fs.conflict_key()` and
  `platform.fs.same_file_path()` to handle case-insensitive filesystems correctly.
- The `make dist` target calls `translate` first; in CI, `pyside6-uic` is run explicitly then PyInstaller is called directly (`.mo` files are in git; avoids cross-platform gettext/bash issues on Windows).
